# 张雪峰 AI 智能体 - 企业级改造日志

## 第一阶段：配置与 Git 清理
- **环境变量管理**：创建 `.env`、`.env.example`，并将 `.env` 加入 `.gitignore`。
- **清理硬编码密钥**：将 `百炼-tools.py`、`ollama调式.py`、`Tools.py` 等实验性及敏感脚本加入 `.gitignore`，并从 Git 仓库的缓存中移除（`git rm --cached`），确保不会提交到代码库。

## 第二阶段：API 服务 P0/P1 风险修复
### P0 风险修复：
1. **API Key 安全**：引入 `python-dotenv`。从 `.env` 或环境变量读取 `DASHSCOPE_API_KEY`，禁用在代码中写入明文密钥。
2. **CORS 收紧**：不再使用 `allow_origins=["*"]`，改为通过环境变量 `CORS_ORIGINS` 配置允许的域名。
3. **接口鉴权**：添加 `API_SECRET_KEY` 配置。配置后，所有请求需要通过 `Authorization: Bearer <key>` 或是 `X-API-Key` 头部进行验证（本地测试可为空）。
4. **会话持久化**：移除基于内存的字典 `_sessions`，替换为基于 SQLite 的 `SessionManager`，支持会话的多实例共享及持久化保存。
5. **输入校验**：使用 Pydantic 对 `message` 和 `session_id` 进行严格输入限制，防止格式注入及超长负荷攻击。字段 `message` max_length 已扩展至 150000 以支持附件文档内容。

### P1 风险修复：
1. **日志系统**：集成标准的 Python `logging` 模块，配置格式化的输出，提供请求开始、结束、异常的具体追踪。
2. **健康检查**：新增 `/health` 接口，提供服务在线状态及基础参数返回（用于负载均衡及 K8s 存活探针）。
3. **异步模型调用**：将底层的模型调用改为异步方式 `ainvoke()` 及 `astream()`，解决原有的同步调用 `invoke()` 阻塞 FastAPI 主事件循环的高并发瓶颈问题。

## 第三阶段：前端与测试
1. **API 地址动态化**：将 `frontend/app.js` 中的常量 `API_BASE = 'http://localhost:8000'` 修改为 `API_BASE = window.location.origin`。确保不管部署在何处都能正常路由。
2. **测试与提交流程**：使用新版 API 服务启动并进行调用验证；验证通过后打包 Git 提交。

## 第四阶段：Agent 智能体升级 + 全网搜索引擎 (V2.0)
1. **ReAct Agent 架构**：底层从简单的 LLM 链式问答重构为 LangChain AgentExecutor 架构，赋予大模型自主反思与工具调用能力。
2. **Tavily Search 集成**：引入 Tavily Search API 作为实时搜索工具，使 Agent 在面对时效性问题（分数线、政策等）时主动联网检索权威数据。
3. **千亿级主力模型**：主引擎切换至 `qwen3.5-397b-a17b`（阿里云百炼），确保极致推理能力。
4. **高可用容灾机制 (Fallback)**：固定配置备用模型 `qwen3.6-35b-a3b`，主模型异常时毫秒级无感切换。
5. **流式事件透传**：使用 `astream_events` 替代 `astream`，精准捕获 Agent 调用工具的中间状态，前端实时显示 `🔍 [老张正在全网检索...]`。

## 第五阶段：万能文档解析 + 多模态视觉 (V2.1)
1. **微软 MarkItDown 引擎**：集成 `markitdown[all]`（含 29 个解析组件），支持 PDF、Word、Excel、PPT、CSV 等格式一键解析为结构化 Markdown。
2. **Qwen3-VL-Flash 视觉模型**：针对图片格式（JPG/PNG/WebP），自动路由至阿里云百炼的 `qwen3-vl-flash-2026-01-22` 视觉模型进行 OCR 提取。
3. **文件持久化存储**：上传的原始文件保存至 `data/uploads/`，并通过 `GET /uploads/{file_id}` 提供下载/预览服务。
4. **前端附件交互优化**：
   - 上传后显示解析进度徽标（`⏳ 正在全自动读取解析...`）。
   - 发送消息时，对话气泡中展示可点击的 `📎 文件名` 链接（浏览器新标签页打开原始文件），而非将解析文本灌入对话框。
   - 发送后输入框旁的附件徽标自动消失，保持界面干净。
5. **时间锚点注入**：每次请求动态注入 `datetime.now()` 到系统提示词，彻底消灭大模型日期/时间幻觉。
6. **全链路异常日志**：注册 `RequestValidationError` 全局拦截器，422 参数校验等框架级报错也写入 `data/app.log`。
7. **ChatRequest 字段扩容**：`message` 的 `max_length` 从 2000 扩展至 150000，支持附件解析内容的透传。

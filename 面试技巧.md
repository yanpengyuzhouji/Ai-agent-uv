# 张雪峰 AI Agent 项目面试复盘指南

这份文档旨在帮助你应对 AI Agent 相关岗位的面试。它基于项目的实际架构，总结了面试官最关心的 10 个核心问题及专业回答参考。

---

## 🚀 核心竞争力概览
- **架构方案**：基于 LangChain 的 ReAct 智能体架构，而非简单的 Prompt Enginering。
- **核心能力**：全网实时搜索（Tavily）+ 万能文档解析（MarkItDown）+ 视觉理解（Qwen-VL）。
- **生产级特性**：高可用 Fallback、流式事件监听、SQLite 会话持久化、参数注入（时间锚点）。

---

## 🎙️ 10 大面试真题与回答锦囊

### 1. 请简述该 Agent 的核心架构，并说明为什么选择 ReAct 模式？
**回答参考：**
该项目基于 **LangChain 的 ReAct (Reasoning and Acting)** 架构。相比传统的简单 Chain，Agent 具有自反思能力。
- **实现逻辑**：模型在接收到问题后，会先产生一个 `Thought`（思考），决定是否需要调用工具（如 Tavily 搜索或文档解析结果）。
- **优势**：它能解决 LLM 知识幻觉和实效性问题。例如问“今年厦大分数线”，模型会主动通过 `Action` 调用 Tavily 获取最新真值，而不是依靠训练数据瞎编。

### 2. 在 Agent 调用工具（Tool Calling）时，你是如何处理流式输出（Streaming）的？
**回答参考：**
这是项目的一大难点。我们使用了 LangChain 的 **`astream_events(version="v2")`** 异步事件流。
- **技术细节**：普通的 `astream` 只能返回最终答案流。`astream_events` 允许我们监听 `on_tool_start` 事件。
- **用户体验**：当监听到工具启动时，后端会推送一个特殊的 `type: content` 消息给前端（如 `🔍 [老张正在全网检索...]`），让用户感知到 Agent 正在工作，避免了长时间的“假死”等待。

### 3. 如何解决大模型对于“当前时间”感知不准确（时间幻觉）的问题？
**回答参考：**
LLM 的记忆停留在训练截止日期。我在 `zhangxuefeng_api.py` 中实现了**动态系统提示词注入**。
- **方案**：在每次 `invoke` 时，通过 Python 获取 `datetime.now()` 并作为参数 `current_time` 注入系统 Prompt。
- **效果**：强制模型以系统提供的时间为物理准绳，有效解决了诸如“今天是几号”、“今年是哪一年”的逻辑推导错误。

### 4. 你的项目支持 PDF/Word/Excel 甚至图片上传，解析流程是怎么设计的？
**回答参考：**
我设计了一套**万能解析网关**：
- **纯文档**：集成微软的 `MarkItDown` 引擎，将 Office 和 PDF 一键转换为 Markdown 格式，因为 Markdown 最适合 LLM 理解。
- **图片/OCR**：如果检测到 JPG/PNG，会自动切换到 **多模态视觉方案**。调用阿里云百炼的 `qwen3-vl-flash` 模型，通过视觉大模型的能力直接提取图片中的文字和表格结构。
- **注入方式**：解析后的内容会作为上下文的前置部分（Context Injection）拼接到用户当前提问的前面。

### 5. 面对超长文档，你是如何处理 Token 溢出风险的？
**回答参考：**
在 `upload_file` 接口中，我实现了 **“防爆截断”** 策略。
- **策略**：在解析出纯文本后，我会根据字符长度进行物理切片（目前限制 10,000 字）。
- **进阶建议**：如果是在生产环境，我会进一步引入 **RAG (Retrieval-Augmented Generation)**。先将文档向量化存入数据库（如 Milvus 或 Chroma），然后根据用户问题仅提取最相关的 Top-K 片段，这样既节省了 Token 成本，又提升了回答精确度。

### 6. 如何保证 API 服务的稳定性和高可用（High Availability）？
**回答参考：**
我实现了一套 **模型回退容灾机制 (Fallback)**。
- **实现**：利用 LangChain 的 `.with_fallbacks()` 方法。
- **逻辑**：将 `qwen3.5-397b` 设置为主模型，同时配置 `qwen3.6-35b` 为从模型。一旦主模型遇到欠费、限流或 500 报错，系统会毫秒级无缝自动切换到从模型继续作答，确保业务永远不中断。

### 7. 在 Web 应用中，如何处理多轮对话的记忆持久化？
**回答参考：**
我并未使用简单的 `Memory` 组件，而是手写了一个基于 **SQLite 的持久化方案**。
- **原因**：内存存储在服务器重启后会丢失，且难以水平扩展。
- **实现**：通过 `session_id` 标识用户，将 `HumanMessage` 和 `AIMessage` 序列化为 JSON 存入数据库。即使服务器宕机重启，用户刷新页面依然能从数据库里找回完整的对话树。

### 8. 为什么前端选择 Native JS 方案而非重量级框架？
**回答参考：**
本项目追求极致的加载速度和交互灵活性。
- **优势**：纯原生 JS 配合 SSE (Server-Sent Events) 能实现最轻量级的流式渲染。
- **创新点**：我们专门优化了附件的展示逻辑——文件解析内容“暗传”给后端，前端气泡只保留一个可点击的“下载/预览链接”，通过这种“数据隔离”保证了 UI 的清爽。

### 9. 服务端报 422 错误且日志里没有记录，你认为是什么原因，如何解决？
**回答参考：**
这是 FastAPI 的参数校验拦截机制。
- **病因**：当上传文档解析后的内容超过了 Pydantic 模型设置的 `max_length` 限制时，FastAPI 会在进入业务逻辑前直接拦截并返回 422 状态码，由于未进入业务层，普通的日志拦截不到。
- **解法**：调整 Pydantic 模型的 `Field(max_length=...)`，并注册全局 `RequestValidationError` 异常处理器，将此类“安检拦截”也强制写入系统日志。

### 10. 如果让你进一步优化这个 Agent，你的下一步计划是什么？
**回答参考：**
这是一个考查潜力的开放题。我会从三个维度进阶：
1. **Agentic RAG**：目前是全文阅读，未来改造成向量检索，支持海量 PDF 资料库分析。
2. **工具链扩展**：接入 Python REPL（代码执行器），让 Agent 能够根据解析出的 Excel 数据自动画出高考分数波动图。
3. **长期记忆（Long-term Memory）**：利用 Redis 或图数据库存储用户的个人画像（如孩子分、省份、兴趣），实现真正的个性化报考方案定制。
